# 按项目收费 & GitHub 权限设置

这份文档回答两个问题：

1) 如果课程按“项目包”收费，怎么组织与对外表达？
2) 如何用 GitHub 给付费学员开权限，同时保护隐私？

---

## 1. 按项目收费：推荐的“项目包”定义

把每个项目当作一个明确交付包：

- 项目目标（学完能做什么）
- 里程碑（每周推进到哪里）
- 交付物（代码、报告、演示、复盘）
- 验收标准（rubric）
- 服务范围（包含几次 review / 答疑窗口 / 允许的延期策略）

建议写法：直接放在对应项目的 README 里（例如 `projects/rag/README.md`）。

---

## 2. 权限与隐私：GitHub 上的三种常用方案

> 结论先说：**一个公开仓库 + 多个私有项目仓库** 通常最清晰。

### 方案 A（推荐）：公开主页仓库 + 私有项目仓库（按项目卖）

- 本仓库（公开）：
  - 展示课程路线、公开资料、每周安排、项目简介
  - 不放：答案、学员信息、收款表、私密课件
- 私有项目仓库：
  - 例如 `ai-course-rag-private`、`ai-course-llm-private`
  - 付费后把学员 GitHub 账号加为协作者或团队成员

优点：

- 权限粒度最符合“按项目收费”
- 哪个项目付费就给哪个仓库权限

如何开权限（个人仓库）：

- Repo → Settings → Collaborators → Add people

如何开权限（组织仓库，更推荐）：

- 创建 GitHub Organization
- 建 Team：`rag-paid`、`llm-paid` 等
- 把私有仓库权限授予对应 Team

### 方案 B：GitHub Classroom（作业发放与回收很强）

适合“每人一份作业仓库/自动收作业”的场景。

- 你创建课堂与作业
- 学员通过邀请链接领取
- 每个学员得到独立仓库（便于验收与防抄作业）

缺点：对“统一给一份资料仓库”不如方案 A 直观。

### 方案 C：全部私有（只对学员开放）

- 主页信息也在私有仓库
- 优点：完全不公开
- 缺点：对外展示与获客不方便

---

## 3. 公共看板（Projects）注意事项

如果你的 Projects 看板是公开的：

- 不要把学员姓名、支付信息、私人会议链接写进条目标题/描述
- 只放“课程主题、日期、备课状态、公开资料链接”

---

## 4. 收款与名单怎么记更安全

- 收款与学员名单建议放在：私有仓库 / 私有 Projects / 私有表格（例如本地或私有云文档）
- 不建议把收款记录放到公开仓库的 README 或 Issues
